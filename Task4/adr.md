#### Обоснование решения:

Решение основано на принципах гибкости и минимального воздействия на текущие процессы. Учитывается обратная совместимость, отсутствие API у партнёров и то, что часть процессов пока не может быть автоматизирована. Бекенд системы внутреннего кол-центра будет опрашивать сервис управления депозитными ставками для получения нового файла с некоторой периодичностью.  

Решение состоит из нескольких компонентов:
1. Система управления ставками
2. Внутренний кол-центр с CRM системой
3. Партнёрские кол-центры с их системами
4. SFTP сервер для обмена файлами
5. Сервис синхронизации
6. REST API для получения данных от сервиса управления ставками
7. База данных для хранения информации по ставкам

Таким образом:
- Файловый обмен выбран для совместимости с существующими системами партнёров
- SFTP для безопасной передачи данных
- Разделение автоматической (внутренний) и ручной (партнёрской) синхронизации
- Уведомления для минимизации задержек обновления

### <a name="_bjrr7veeh80c"></a>**Альтернативы**

#### Непосредственная API-интеграция

Рассматривался подход интеграции систем партнерских кол-центров с API-шлюзом для получения информации по ставкам

**Преимущества**

1. Исключается ручной подход
2. Нет необходимости в передаче файлов по SFTP

**Недостатки, ограничения, риски**

1. Политика безопасности. Использование такого API во внешнем контуре требует более тщательной проработки и согласования отделом ИБ
2. Невозможность реализации на стороне партнёрского кол-центра в виду ограниченных технических возможностей. Требуются изменения в системе партнёрского-кол-центра 

#### Использование существующей (внутренней) CRM системы для всех участников

Рассматривался подход предоставления доступа к существующей CRM системе банка для партнерских кол-центров

Преимущества

1. Использование готового решения
2. Единые процессы работы со ставками

Недостатки, ограничения, риски

1. Критические риски информационной безопасности при предоставлении внешнего доступа к внутренней CRM
2. Необходимость существенной доработки системы разграничения доступа
3. Высокие лицензионные затраты на дополнительных пользователей
4. Сложность администрирования
