|**№**|**Требование**|

|NFR5|Интеграция: REST API для взаимодействия с внешними системами|
|NFR6|Мониторинг: Логирование всех операций и метрики производительности|
|NFR7|Отказоустойчивость: Graceful degradation при недоступности внешних систем|
|NFR8|Кэширование: Кэширование предложений для снижения нагрузки на внешние системы|

#### Обоснование решения
Выбрана микросервисная архитектура с разделением на специализированные сервисы для обеспечения масштабируемости и независимости развертывания.


Решение состоит из нескольких компонентов:
1. Сервис управления кредитами: управление заявками, получение информации о предварительно одобренных кредитах, предложения по ставкам
2. Сервис предварительно одобренных кредитов: массовый расчёт предодобренных предложений по клиентам И планировщик запуска таких задач
3. База данных кредитов

Сервисы интегрированы с уже существующими Кредитным конвейером и АБС.

- Микросервисная архитектура выбрана для обеспечения независимого масштабирования компонентов
- Данные по предварительно одобренных кредитам хранятся в MS SQL, что ускоряет процесс обработки заявок и снижает нагрузку на Кредитный Конвейер
- REST API обеспечивает простоту интеграции с существующими системами банка

### **Альтернативы**

1. **Монолитная архитектура**
   - Проще в разработке и развертывании
   - Меньше сложности в управлении
   - Но ограничена в масштабируемости и гибкости

2. **Event-driven архитектура**
   - Высокая отказоустойчивость
   - Лучшая производительность для асинхронных операций
   - Но сложнее в отладке и мониторинге

### **Недостатки, ограничения, риски**

**Недостатки:**
- Устаревающий подход в виде файлового обмена
- Зависимость от polling-а новых файлов

**Ограничения:**
- Отсутствие автоматизации у партнерского кол-центра
- Неуниверсальный формат данных (XLS/CSV)

**Риски:**
- Возможные уязвимости SFTP-обмена
